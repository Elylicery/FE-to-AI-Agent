## 1. RAG基础

### 1.1 幻觉原因及缓解方案
#### 1.1.1 幻觉

##### **幻觉的定义**

回答不准确、前后不一致问题，大概分为两类：
1. 事实性幻觉：与现实世界信息矛盾、事实捏造
2. 忠实性幻觉：模型生成内容与用户指令或上下文不一致，例如指令不一致（输出偏离用户指令）、上下文不一致（输出与上下文信息不符）、逻辑不一致（推理步骤以及最终答案之间不一致）

##### **产生幻觉的原因**

- 数据源导致：
    - 数据缺陷：包含错误信息和偏见（重复偏见、社会偏见）
    - 知识边界：存在领域知识缺陷和过时事实知识
    - 模式依赖：过度依赖训练数据中的位置接近性、共现统计等模式。例如频繁出现"加拿大"和"多伦多"可能导致模型错误认为多伦多是加拿大首都
- 训练过程导致：
    - 预训练阶段：
        - 架构缺陷：单向建模限制上下文理解能力
        - 暴露偏差：错误token产生级联错误
    - 对齐阶段：
        - 能力错位：生成超出知识边界内容
        - 信念错位：为迎合人类偏好牺牲真实性
- 推理导致：
    - 抽样随机性：基于概率随机生成
    - 解码缺陷：上下文关注不足和softmax瓶颈


##### **幻觉评估**

* 事实性幻觉评估：
    - 检索外部事实：将生成内容与可靠知识源对比
    - 不确定性估计：
        - 基于内部状态：分析关键概念的最小标记概率
        - 基于行为：采样多个响应评估一致性
- 忠实性幻觉评估：
    - 基于事实的度量
    - 分类器度量
    - 问答度量
    - 不确定度估计
    - 提示度量
#### 1.1.2 缓解方案
##### ⭐缓解数据相关幻觉

方法：高质量数据：收集事实数据并进行清理，两种方法：
- 知识编辑：直接编辑模型参数弥合知识差距
- ⭐检索增强生成(RAG)：
    - ⭐一次性检索：单次检索知识直接预置到提示中
    - 迭代检索：生成过程中不断收集知识
    - 事后检索：基于检索修订完善输出
![[images/Pasted image 20260713162735.png]]

##### 缓解预训练相关幻觉

1. 改进模型架构：使用双向自回归模型和注意力锐化技术
2. 优化训练目标：引入事实性增强训练方法
3. 减少曝光偏差：采用新的监督信号和解码策略

##### 缓解对齐相关幻觉

1. 减少能力错位：确保生成内容在知识范围内
2. 减少信念错位：平衡人类偏好与事实真实性

##### 缓解推理相关幻觉

1. 增强事实性解码：动态调整解码策略
2. 增强忠实度解码：确保输出与指令保持一致

#### 1.1.3 检索增强生成RAG

检索增强生成(RAG)是对大型语言模型(LLM)输出的优化技术，使其能在生成响应前引用训练数据之外的权威知识库。通过数十亿参数的LLM结合外部知识，无需重新训练模型。

运行机制：**从外部检索相关知识内容，与用户提问共同构成Prompt，再由LLM生成内容**。

### 1.3 文本嵌入模型与RAG开发

#### 1.3.1 Embedding

**Embedding：**
一种模型生成方法，可以将非结构化的数据，例如文本/图片/视频等数据映射成有意义的向量数据**
![[images/Pasted image 20260714210127.png]]
生成embedding方法的模型有：
1. NLP领域的 word2Vec Glove、FastText
2. ⭐**大模型Embeddings（重点）**

**Embedding带来的价值：**
 1. **降维**：高维数据映射到低维空间，减少模型复杂度
 2. **捕捉语义信息**：保留原始数据的一些重要信息
 3. **适应性**：数据驱动的方式学习，自动适应数据的特性
 4. **泛化能力**：处理一些训练数据中没有出现过的数据。也可以捕获数据内在规律
 5. **可解释性**：可以通过可视化工具（如 t-SNE）观察和理解embedding的结构

“语义上相近的词在向量空间中也会相似“

#### 1.3.2 Enbedding模型

1. OpenAI的text-embedding-3 系列，包含：text-embedding-3-small、text-embedding-3-large
2. Hugging Face本地模型、Hugging Face远程嵌入模型
3. 国内：BGE、Qwen、百度千帆等

- 余弦相似度：
    - 原理：计算向量夹角余弦值，公式：

    - 特点：值域[-1,1]，值越大越相似；适用于文本、图像等高维数据，忽略向量长度。
- 欧式距离：
    - 原理：计算向量直线距离，公式：![[images/Pasted image 20260713223525.png|221]]
    - 特点：值最小为0，越小越相似；适合低维数据，考虑各维度差异。
##### 相似度搜索加速计算

加速算法：

1. 矩阵分解：SVD（奇异值分解）和PCA（主成分分析）降维减少计算量。
2. 索引优化：LSH（局部敏感哈希）和ANN（近似最近邻）算法加速搜索。
3. 硬件加速：利用GPU并行计算提升矩阵运算速度。
4. 分布式计算：因行间独立性强，适合分布式处理。
5. 存储优化：列式存储结合稀疏矩阵压缩，提升IO效率
### 1.4 向量数据库

#### 1.4.1 向量数据库

* **向量**：任何事物（文字、图片等）均可转换为多维向量，文本转换后的特征向量称为词向量。
* **向量数据库**：专门优化存储/查询高维向量数据，解决传统关系型数据库处理高维数据效率低下的问题。
* **相似度计算原理**：通过向量距离衡量相似度
- 核心应用场景：
    - 人脸识别：归一化人脸图像→生成向量→数据库比对
    - 图像/音频搜索：基于特征向量相似性检索
    - 智能推荐：相似内容向量聚类推荐
- RAG中的应用：
    - 知识文档→分块向量化存储
    - 提问query→向量化搜索→返回最相关文本块
    - LLM结合补充知识生成回答

#### 1.4.2 关系型数据库与向量数据库

**搜索差异：**

相似性搜索：向量数据库通过**计算向量距离实现近似匹配**，例如即使没有完全相同照片也能找到风格相似的图片，支持在相似性搜索基础上添加筛选条件。

**数据处理与存储差异：**

向量数据库将**每列单独存储，支持稀疏矩阵压缩**，通过矩阵运算优化提高性能

**对比表格：**

| 维度      | 传统的关系型数据库                                   | 向量数据库                                             |
| ------- | ------------------------------------------- | ------------------------------------------------- |
| 数据类型    | 数值、字符串、时间等传统数据类型                            | 新的数据类型：向量数据；不存储原始数据（有的也支持）                        |
| 数据规模    | 小，1亿条数据对于关系型数据库来说规模很大                       | 大，最少千亿数据是底线                                       |
| 数据组织方式  | 基于表格，按照行和列组织                                | 基于向量，按照向量维度组织                                     |
| 查找方式    | 精确查找：点查/范围查；查询结果要么符合条件要么不符合条件               | 近似查找；查询结果是与输入条件最相似的，近似计算能力要求较高                    |
| 低延时，高并发 | 否                                           | 是                                                 |
| 上层应用    | 较弱                                          | 对外提供统一的API，更适合大规模AI应用程序的部署和使用                     |
| 下游应用场景  | 央企、国企。央企国企工作因工作内容要求容错率低，传统数据库能够提供更为准确的搜索结果。 | 互联网公司。向量数据库的结果正确率相对较低，成本低，互联网公司的场景容错率较高，能够包容这一缺陷。 |


#### 1.4.3 Faiss 本地向量数据库

facebook团队的开源向量检索工具。数据存储在**本地文件系统**，通过数据库接口检索，适合轻量级应用场景。

 基础操作：支持数据新增、检索（含得分检索）、条件筛选和删除，但不支持直接修改。

   **向量数据库不支持修改数据的原因：**
- 索引结构：使用向量索引树或近似最近邻算法，数据插入后需要构建优化索引。
- 性能考量：直接修改会导致索引频繁重建，显著增加系统开销，故采用"删除+新增"策略
#### 1.4.4 云端向量数据库
